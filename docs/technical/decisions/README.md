# Decisions

## Cel folderu

Ten folder przechowuje ADR-y, czyli Architecture Decision Records.

## Kiedy tworzyc ADR

ADR jest potrzebny, gdy decyzja:

- ma koszt zmiany,
- ma wiecej niz jedna sensowna alternatywe,
- wplywa na wiele czesci systemu,
- bedzie wracac w przyszlosci podczas utrzymania.

## Konwencja

- `ADR-001-nazwa-decyzji.md`
- `ADR-002-nazwa-decyzji.md`

Statusy:

- `proposed`
- `accepted`
- `superseded`
- `deprecated`

## Pierwsze kandydaty na ADR

- strategia bazy danych,
- strategia i18n,
- strategia auth,
- granice CMS i dynamicznych sekcji.
